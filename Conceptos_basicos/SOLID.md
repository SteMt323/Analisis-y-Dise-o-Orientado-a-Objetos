# Los Principios de SOLID

| Letra | Principio                      | Descripción                                                                 |
|------|--------------------------------|-----------------------------------------------------------------------------|
| S    | Responsabilidad Única           | Una clase debe tener un solo propósito y una sola razón para cambiar.      |
| O    | Abierto / Cerrado              | El código debe estar abierto a extensión (añadir funcionalidad), pero cerrado a modificación (alterar código existente). |
| L    | Sustitución de Liskov          | Las subclases deben poder sustituir a su clase base sin romper el comportamiento del sistema. |
| I    | Segregación de Interfaces      | Es mejor tener múltiples interfaces específicas que una interfaz general y monolítica. |
| D    | Inversión de Dependencias      | Depender de abstracciones (interfaces), no de implementaciones concretas.  |
