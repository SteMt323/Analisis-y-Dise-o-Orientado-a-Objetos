## Dominio del Problema
Conjunto de conceptos, reglas y entidades del mundo real que el sistema debe representar. Permite entender que problema se quiere resolver y evita ambiguedades.  

### Relaciones entre clases
* **Asociacion**:  Relacion de *conocimiento*. Una clase usa a otra de forma independiente.

* **Composicion**:  Relacion de *propiedad fuerte*. Si el contenedor desaparece, el contenido tambien.

* **Herencia**: Relacion de *jerarquias*. Permite reutilizar atributos y metodos.

## Caso de Usos
Conjunto de secuencia dea cciones que se ejecutan, y cuyo resultado es de interes para un actor en particular.

**Actores**: Entidad externa que interactua con el sistema. Ellos representan a todo lo que necesita  intecambiar infroamcion con el sistema.
